## Lessons Learned

1. **Ghost trades are a distributed consistency problem**: When MT5 closes a position between oracle cycles, the DB doesn't know. Reconciliation (compare DB vs MT5 state + deal history) is the correct fix, not just flagging `is_open=0`.
2. **Always set exit_price**: A trade with `is_open=0` but `exit_price=NULL` is worse than no record — it corrupts PnL calculations and win rate metrics.
3. **Module-level logger pattern**: Python module-level `logger = logging.getLogger(__name__)` is the standard. Function-scoped loggers work until a new function outside that scope tries to use them.
4. **RPyC limitations are real**: No datetime serialization, no **kwargs forwarding, no .get()/.keys() on netref dicts. Always use positional args, Unix timestamps, and bracket access.
