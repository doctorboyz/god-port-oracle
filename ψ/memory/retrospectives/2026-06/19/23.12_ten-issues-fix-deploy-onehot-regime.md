## Architecture Decisions

1. **One-hot > ordinal สำหรับ regime** — Ordinal encoding (trending=1, ranging=0, volatile=2) บอก ML ว่า volatile "มากกว่า" trending ซึ่งไม่จริง. One-hot ทำให้แต่ละ regime เป็น independent feature
2. **Backward compat** — เก็บ regime_encoded ไว้สำหรับ v4 models ที่ train ด้วย ordinal, เพิ่ม one-hot สำหรับ v6+
3. **Dead code removal over deprecation** — PersistentMT5Bridge ไม่มี caller แล้ว, ลบออกเลยดีกว่าปล่อยไว้ให้ confuse

## Lessons Learned

1. **One-hot encoding > ordinal สำหรับ categorical features ที่ไม่มี order** — Regime (trending/ranging/volatile) ไม่มี natural order. Ordinal บอก model ว่า volatile > ranging > trending ซึ่งผิด. One-hot ทำให้แต่ละ category เป็น independent binary feature.

2. **Dead code removal > deprecation** — ถ้าไม่มี caller อยู่แล้ว, ลบเลยดีกว่าปล่อยไว้. PersistentMT5Bridge 145 บรรทัด ไม่มีใครเรียก แต่ยังอยู่ใน pre-deploy check → confusion.

3. **Transition features ต้องมี exit plan** — ตอนนี้มีทั้ง ordinal + one-hot อยู่ด้วยกัน. ต้องมีกำหนดว่าเมื่อไหร่จะเอา ordinal ออก (ตอน v4 retire).
