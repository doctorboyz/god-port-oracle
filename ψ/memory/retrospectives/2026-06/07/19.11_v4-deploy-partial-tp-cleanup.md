## Lessons Learned

1. **Feature pipeline ต้องมี single source of truth**: เพิ่ม feature ใหม่แล้วลืมอัปเดต feature list ทุกที่ = bug ที่ซ่อนอยู่ ควรมี `_FEATURE_COLUMNS` constant เดียวและ derive ทุกอย่างจากนั้น
2. **v4 > v5 เพราะ prediction diversity**: model ที่ส่งค่า varied กว่า (ไม่ constant ~0.78) แยก loss trades ได้ดีกว่า
3. **Data gap ใน historical features**: `h4_trend` ไม่อยู่ใน `features_json` ของ trades เก่า → ต้องสะสม live data ก่อน retrain
4. **tp1_ratio hardcoded เป็น anti-pattern**: parameter ที่ config ได้ต้องอ่านจาก RiskConfig เสมอ ไม่ใช่ magic number
