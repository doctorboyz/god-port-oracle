## Key Code Changes

### `get_connection()` — Accept str and Path
```python
# Before: crashed on string paths
def get_connection(db_path: Optional[Path] = None) -> sqlite3.Connection:
    path = db_path or DB_PATH  # str has no .parent → crash

# After: converts str to Path
def get_connection(db_path: Optional[Path | str] = None) -> sqlite3.Connection:
    path = Path(db_path) if isinstance(db_path, str) else (db_path or DB_PATH)
```

## AI Diary

I'm honestly frustrated with myself that it took 4 VPS deploy cycles to get this right. The first issue — `get_connection()` not accepting string paths — was something I should have caught when writing the function. The second issue — backfill skipping existing rows instead of updating them — was a design oversight. I was thinking "backfill" means "insert missing rows" but forgot that existing rows might have NULL columns that need updating too.

The third issue was the most embarrassing: I ran the backfill on `trade_outcomes` assuming `live_trades` already had the data, but it was NULL all the way down. The backfill can only copy from source, it can't invent values. I had to manually UPDATE `live_trades` based on `account_id` mappings.

What I'm taking away: when doing data migrations, always verify BOTH the source and target tables have complete data. A pipeline that copies from A→B is useless if A is empty.

## What Could Improve

1. **4 deploy cycles for one fix** — each cycle is 2-3 minutes of build + restart. Should have caught all issues in one pass by checking the full data chain before deploying
2. **No automated data integrity check** — there's no test that verifies "every row in trade_outcomes has trading params populated". Should add a data quality check
3. **String vs Path type inconsistency** — `get_connection()` only accepted `Path` but callers sometimes pass strings. Should standardize types across the codebase

## Honest Feedback

1. **Deploy velocity is too slow** — 4 cycles × 3 min build = 12 min just waiting. Should batch fixes or add a local DB test that mimics VPS schema before deploying
2. **Type safety is weak** — Python's dynamic typing let a `str` vs `Path` bug slip through silently. The function signature said `Optional[Path]` but nothing enforced it. Should add type guards or runtime checks at DB boundaries
3. **Data migration strategy is ad-hoc** — I'm running raw SQL on production DB via SSH. No migration scripts, no rollback, no audit trail. One wrong UPDATE could corrupt 10K+ rows

## Lessons Learned

1. **Data chains must be verified end-to-end** — When copying A→B, verify A has data first. Backfill is only as good as its source
2. **Existing rows need UPDATE, not just INSERT** — Backfill shouldn't mean "insert missing" only; it should also mean "update incomplete"
3. **Account-based param mapping** — Since each account has fixed trading params, we can always reconstruct missing data from account_id
