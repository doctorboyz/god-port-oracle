## Honest Feedback

1. **Scope blindness**: I changed `self._last_h4_trend` to `h4_trend` in one line but didn't trace where `h4_trend` comes from. The audit verified the line looked right but couldn't detect that the variable was in the wrong scope. This is a pattern — fixing references without verifying definitions.

2. **No integration test for DB writes**: The INSERT mismatch only appeared in production because we don't have a test that exercises the full trade insertion path. A simple `test_insert_live_trade_all_columns()` would catch this immediately.

3. **Docker rebuild is slow**: 60+ seconds per deploy cycle makes the fix-deploy-verify loop feel glacial. The `--build` flag rebuilds everything even when only one Python file changed.

## Lessons Learned

1. **Scope verification**: When changing a variable reference (A→B), always verify B is defined in the same scope where it's used. Different methods have different local scopes.
2. **SQL placeholder counting**: Always count `?` placeholders against column names in INSERT statements. A linter or pre-commit hook could catch this.
3. **Production-only bugs**: Some bugs only surface in production (DB writes, MT5 order execution) because dry-run paths skip those code paths. Need integration tests that exercise the full path.
