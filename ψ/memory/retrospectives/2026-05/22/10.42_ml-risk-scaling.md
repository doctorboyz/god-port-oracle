## Architecture Decisions

1. **Risk-scaling แทน hard blocking**: เพราะ underlying model signal อ่อนเกินไปสำหรับ binary decision — calibration พิสูจน์แล้วว่า degrade performance — risk-scaling ให้ model มีประโยชน์แม้ signal จะ weak
2. **keep should_skip() ไว้**: backward compatibility — ถ้าอยากกลับไป hard blocking ก็ทำได้
3. **minimum lot size guard**: ถ้า multiplier ทำ lot < 0.01 → skip trade เพราะ Exness ขั้นต่ำ 0.01

## Honest Feedback

1. **Calibration was a dead end we should have seen sooner**: ใช้เวลา 2-3 iterations กับ isotonic/sigmoid calibration ทั้งที่ข้อมูลไม่พอ (300-500 samples ต่อ model) — isotonic regression บน 3-fold CV แทบไม่มีทางทำงานได้ดี ควรรู็ตั้งแต่แรกว่าต้องการอย่างน้อย 1000+ samples

2. **trainer changes ไม่ได้ deploy**: เพิ่ม scale_pos_weight ใน trainer แต่ไม่ได้ train v5 แล้ว deploy — ตอนนี้ VPS รัน v2 model ที่ไม่มี scale_pos_weight, risk-scaling ใช้ probability จาก model ที่ไม่ได้ถูก train ด้วย class balancing — อาจจะดีกว่า v2 เดิม แต่ก็ยังไม่ optimal

3. **Risk threshold 0.50/0.85 คือ arbitrary**: ไม่ได้ backtest หา optimal threshold — เลือกจาก intuition ล้วนๆ — 0.50 คือ breakeven point ของ binary classification, 0.85 คือ conservative cutoff — แต่ไม่มีหลักฐานว่า thresholds พวกนี้ดีที่สุด

## Lessons Learned

- Calibration (isotonic/sigmoid) ต้องการ samples มากกว่าที่มี — 500 rows ไม่พอสำหรับ Platt scaling หรือ isotonic regression
- Risk-scaling เป็นทางเลือกที่ดีเมื่อ model signal weak — ใช้ประโยชน์จาก probability ได้โดยไม่ต้อง binary decision
- XGBoost probability distribution มัก bimodal (ใกล้ 0 หรือ 1 มาก) — scale_pos_weight ช่วยได้บ้าง แต่ไม่เปลี่ยน fundamental distribution
