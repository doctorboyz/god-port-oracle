## Lessons Learned

1. **Bridge connection overhead**: Creating separate MT5Bridge instances for each operation (fetch, spread, order) causes rapid connect/disconnect cycles that may contribute to order failures. Reuse connections within a cycle.

2. **Ticket prefix = server identity**: Exness tickets starting with 1902... are from MT5Trial17 (accounts A & B), 4006... from MT5Trial7 (account C). "Seeing C trades in A's VNC" is a confusion, not contamination.

3. **Learning mode risk**: Without position limits, learning mode opens unlimited positions. Account A went from $100 to $46.92 equity. Need per-account max position limits.

4. **Phantom trades**: When orders fail (ticket=None), the trade is still recorded as open in the DB. Need to either not insert failed orders or immediately close them with a reason.
