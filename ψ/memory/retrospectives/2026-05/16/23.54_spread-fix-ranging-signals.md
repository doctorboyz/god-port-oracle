## Key Code Changes

### Root cause: MT5 spread data

`mt5.symbol_info()` returns **static** symbol properties where `spread` defaults to 0. The actual market spread is `(ask - bid) / point` from `mt5.symbol_info_tick()`. The bridge had no method to expose tick-based spread — added `get_spread_sync()` and async `_get_spread()`.

### Ranging signal generation

When ADX < 20, instead of returning HOLD, the generator now routes to `_generate_ranging_signal()`:
- Bollinger Bands primary: price ≥ 70% toward band → mean-reversion signal
- MACD mild confirmation (0.30 weight, not primary)
- Volume bonus/penalty (±0.10-0.15)
- Confidence capped at 0.65 (ranging signals inherently weaker)
- Session filter still applies post-signal

## Lessons Learned

### spread-static-vs-tick
`mt5.symbol_info().spread` is a static reference value (0 by default), NOT live market spread. To get actual spread: `(symbol_info_tick.ask - symbol_info_tick.bid) / point`. Never use symbol_info for runtime spread filtering.

### ranging-can-be-profitable
ADX < 20 ≠ "don't trade." Our live data shows ranging WR 66.7% vs trending 46.2%. Ranging requires different strategy (mean-reversion via Bollinger) but is NOT unprofitable. Don't let textbook assumptions override your own data.

### signal-monitoring-gap
Without per-strategy signal rate monitoring, a strategy can silently fail for 10+ days in production. Need a dashboard or at least a periodic health check that logs signal counts per strategy per day.
