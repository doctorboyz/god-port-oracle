## Honest Feedback

1. **Trend filter asymmetry ทำพอร์ตเจ็บ**: D1 bullish → block SELL, allow BUY. แต่ในตลาดที่กลับตัวแรงๆ BUY ก็ควรถูกระวังเหมือนกัน — เราไม่มี safety net สำหรับ "D1 bullish แต่ราคาลงแรง" ทั้งที่ควรมี

2. **Deploy ไปคราวก่อนไม่ครบ**: `get_risk_multiplier()` มีใน local แต่ไม่มีบน VPS — ทำให้ swing trader crash loop เงียบๆ โดยที่เราไม่รู้ เพราะ error ถูก swallow หรือ log ไม่ชัดพอ ควรมี health check ที่ verify ว่า ML filter ทำงานจริง ไม่ใช่แค่โหลด model แล้วเงียบ

3. **ML bias จาก training data**: v2.best models เทรนจากช่วง D1 bullish ซะส่วนใหญ่ เลย predict P(LOSS) สูงมากกับ SELL ทุกอัน — แม้แต่ตอนนี้ที่ D1 bearish แล้ว ML ก็ยังบล็อก SELL อยู่ดี ต้องมี regime-aware retraining

## Lessons Learned

- **D1 EMA 50/200 ช้าเกินสำหรับการกลับตัว**: ราคาลง -$73 (1.3%) กว่า D1 จะยอมรับว่าเป็น bearish — H4 กลับตัวเร็วกว่า 4 เท่า แต่ก็ยังช้า (~20 ชม.) การมี price momentum filter (เช่น ราคาลง >1% ใน 24h → ลดความเชื่อมั่นของ D1) น่าจะช่วยได้
- **Silent crash = silent losses**: ML filter crash แล้ว swing trader เทรดต่อโดยไม่มี filter — ต้องมี circuit breaker: ถ้า ML filter failed > N ครั้งติด → stop trading
- **D1 flip เป็น event สำคัญ**: ระบบเปลี่ยนพฤติกรรมเองโดยไม่ต้องแก้อะไร — จาก BUY-only เป็น SELL-allowed — architecture ดีที่ adapt ได้ แต่การไม่มี alert สำหรับ trend change ทำให้เราไม่รู้จนมาเช็คเอง
