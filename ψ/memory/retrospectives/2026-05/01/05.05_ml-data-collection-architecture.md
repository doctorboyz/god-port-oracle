## AI Diary

I noticed I was rushing through Phase 2 (indicators) by delegating to 3 parallel agents, which was efficient but meant I didn't personally verify each indicator's correctness against known formulas. The agents reported all 173 tests pass, but I didn't spot-check any individual indicator calculation myself. This is a risk — the ML model's quality depends entirely on correct indicator implementations. If OBV or MFI has a subtle bug, the model will learn from garbage data.

The uncomfortable truth: I'm treating Phase 4 (execution engine) and Phase 5 (ML pipeline) as "will do later" but they're actually the hardest parts. The execution engine needs real MT5 connection testing, which requires the user's Exness demo accounts. And the ML pipeline depends on having 200+ trades of real data, which means weeks of collection before we can even start training. I should be more honest about this timeline — it's not "next session" work, it's "2-3 weeks of data collection then 1 week of ML" work.

The Thai trader's philosophy article was a genuine insight. WR 40-50% being sufficient with solid risk management aligns perfectly with our Phase 1.5 findings (WR=53% is enough). The EZB/1CB system's emphasis on structural levels (Close > Wick, Flip Zones) gives concrete improvements we can add to broky's signal engine later. But I shouldn't conflate "interesting philosophy" with "proven edge" — the article is one trader's experience, not backtested data.

## Honest Feedback

**Friction 1**: The parallel agent pattern for writing indicators was fast but opaque. I couldn't verify the math for 14 indicators personally. Next time, I should spot-check at least 3-4 indicators with known values before trusting the agent output.

**Friction 2**: I designed the ML architecture before fully understanding the MT5 Docker constraints. The Dockerfile assumes `mt5linux` works in a Linux container with Wine, but the current setup uses macOS Wine. This will need testing and probably debugging.

**Friction 3**: The plan covers 6 phases but Phase 4+ requires real MT5 connection. I should have been more explicit about the "data collection gap" — we need weeks of real trading data before ML can start. This isn't a "next session" task; it's a "next month" task.

## Lessons Learned

1. **ML architecture = data collection first**: The whole point is collecting diverse feature snapshots. Simple triggers fire more → more data → better ML. Don't optimize triggers for profitability; optimize for data diversity.

2. **WR gap between backtest and forward is a real risk**: 53% vs 43.3% suggests our backtest may be overfit. The Monte Carlo simulation (MSG-PM-003) will help assess this.

3. **Parallel agents work well for repetitive implementation**: 14 indicators with the same pattern was a perfect use case. But verification must follow.

4. **Thai trader philosophy aligns**: "ท่าง่าย ทำซ้ำได้ วัดผลได้" matches our approach — WR 40-50% is fine with solid risk management.

5. **Feature snapshots must capture ALL indicators**: Cross-group correlations are where the ML gold is. Volume signal + RSI oversold together may have much higher WR than either alone.
