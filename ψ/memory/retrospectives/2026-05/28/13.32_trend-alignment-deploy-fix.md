## Honest Feedback

1. **No pre-deploy safety net**: Syntax check + import test ควรเป็นขั้นตอนบังคับก่อน deploy ทุกครั้ง ตอนนี้มันขึ้นอยู่กับว่าคนเขียนจำได้ไหม — และ session ก่อนเราลืมเพราะ context หมด การไม่มี automated check ทำให้ deploy โค้ดที่พัง syntax ด้วยซ้ำ

2. **Decorator with import-time side effects = landmine**: `@strategy()` register strategy ตอน import ทำให้การเขียน utility function ใหม่ใต้ decorator เก่าเป็น fatal error ทันที ไม่ใช่แค่ logic bug แต่ import ทั้ง module พัง — ถ้าเราใช้ decorator ที่มี side effect ควรมี test ที่ verify ว่า import chain ทำงานได้

3. **Session compaction hides unfinished work**: โค้ดถูกเขียนแต่ไม่ verify แล้ว session จบ — compaction ทำให้ context กลับมาแบบ "incomplete" แต่เราไม่รู้ว่าอะไร incomplete จนกว่าจะรัน การมี checklist ก่อน session จบ (syntax check, import test, basic smoke) จะป้องกันปัญหานี้ได้

## Lessons Learned

- **Decorator ≠ ปลอดภัย**: `@strategy` มี side effect ตอน import — utility function ไม่ควรมี decorator ที่ register มันเป็น signal generator การ refactor โค้ดใต้ decorator ต้องเช็คว่า decorator ยัง apply กับ function ที่ถูกต้อง
- **Pre-deploy syntax check เป็น mandatory**: ก่อน deploy ทุกครั้งต้องผ่าน `python3 -c "import py_compile"` กับทุกไฟล์ที่แก้ — ไม่ใช่ nice-to-have, เป็น requirement
